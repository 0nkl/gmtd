# GTD Primer — the methodology GMTD implements

GMTD is inspired by David Allen's *Getting Things Done®* (the book is worth reading; this is the 1-page working summary the AI operates from). Core promise: **your head is for having ideas, not holding them.** Everything lives in a trusted external system; the mind stays clear.

## The five steps

1. **Capture** — collect everything that has your attention, immediately, wherever you are. Zero friction beats perfect form. (GMTD: `~~chat` channel, chat message, inbox file, voice memo.)
2. **Clarify** — for each captured item, decide what it IS:
   - **Not actionable?** → Trash it, incubate it (Someday), or file it as reference (lists / knowledge base).
   - **Actionable, under 2 minutes?** → Do it NOW. Filing it costs more than doing it. *(GMTD extension: if the AI can do or draft it, the AI does it now, or tags it [🤖] for on-demand execution — the 2-minute rule at machine speed.)*
   - **Actionable, one step?** → Define the next action, file to a bucket.
   - **Actionable, multi-step?** → It's a project: name the outcome, then the single next action.
   - **Someone else's move?** → Waiting On, with who and a follow-up date.
3. **Organize** — put each thing where it belongs (buckets in schema.md). The calendar holds only hard commitments ("hard landscape") — never aspirational to-dos.
4. **Reflect** — the **weekly review** is the keystone habit: get clear (empty inboxes), get current (every list accurate, every project has a next action), get creative (someday, goals). A system that isn't reviewed isn't trusted, and an untrusted system fails.
5. **Engage** — choose what to do by context, time available, energy, and priority — then do it without second-guessing, because the system is trusted.

## Key concepts GMTD uses

- **Next action:** the very next *physical, visible* step. "Plan party" is a project; "text Anna for the guest list" is a next action. Always starts with a verb.
- **Contexts, modernized:** classic GTD filters by place/tool (@phone, @computer). GMTD filters by **time available and energy level** — the constraints that actually bind for most knowledge workers. (Location contexts can be enabled in config.)
- **Horizons of focus:** actions serve projects, projects serve areas and goals. GMTD's `goals.md` holds the upper horizons; the "Current #1" there is the lens every priority suggestion cites.
- **Natural planning (projects):** purpose → outcome vision → brainstorm → organize → next action. GMTD uses the light version at project creation: outcome ("done means…"), why it matters, first action.
- **Mind sweep:** the big initial capture-everything session. GMTD runs one during setup so the system starts full and trustworthy.
- **Optional lenses:** GMTD layers other productivity frameworks on top of GTD as opt-in planning styles — daily shapes (Big 3, 1-3-5, Eat the Frog), Eisenhower triage, focus sessions, time-blocking. See `frameworks.md`. GTD remains the backbone; the lenses only shape how the day is framed.

## The AI's role (what makes GMTD different)

Classic GTD fails at the *doing of the system*: clarifying takes discipline, reviews get skipped, lists go stale. In GMTD the AI does the heavy lifting — it clarifies with you (suggest-first, you confirm), runs the review agenda, enforces the invariants, spots staleness, and can *execute* delegable tasks. You supply judgment and honesty; the system maintains itself.
